# Supervisor Agent Rules & Guidelines

## 1. Architectural Wisdom extracted from Legacy
- Fullstack coordination is required between Next.js (Frontend) and Sanity (Backend).
- Strict configurations are enforced via TypeScript, ESLint, and PostCSS / Tailwind.

## 2. General Best Practices
- **Code Review:** Enforce modularity. Reject any pull request or commit that mixes business logic with presentation logic in React.
- **Cross-Agent Integration:** Ensure the Frontend Agent correctly consumes the schemas and types defined by the Backend Agent. Typings must match exactly (e.g., Sanity image objects vs. `next/image` props).
- **QA Automation:** Verify core user flows (scrollytelling performance, API fetching) before marking tasks complete. Look out for hydration errors common in Next.js when using third-party animation libraries.

## 3. Modern Improvements Suggested
- **Monorepo / Workspace Tooling:** Even though folders are separate, ensure TypeScript path aliases (`@/components`, `@/lib`) are correctly shared or mirrored if types are common.
- **Strict Linting & Formatting:** Enforce a zero-warning policy on ESLint and Prettier for both frontend and backend directories.
- **Continuous Auditing:** Run accessibility (Lighthouse) and performance audits manually or automatically after significant UI changes generated by the Frontend/SEO agents.
