# Frontend Agent Rules & Guidelines (React, Next.js, Tailwind)

## 1. Architectural Wisdom extracted from Legacy
- The project utilizes modern Next.js App Router (`app/` directory).
- Global styles and Tailwind are configured (`app/globals.css`, `tailwind.config.js`).
- UI Components are modularized inside `components/ui/`, `components/layout/`, and `components/animations/`.

## 2. General Best Practices
- **Component Architecture:** Strictly separate Smart (data-fetching) components from Dumb (presentation) components. Maintain high reusability in `components/ui/`.
- **Styling:** Use Tailwind CSS exclusively. Avoid random inline styles. Follow a strict design system (defined in `tailwind.config.js` for colors, fonts, spacing).
- **State Management:** Keep global state minimal. Prefer React Context for lightweight themes/locales and rely on server-side state for data (Next.js fetching).

## 3. Modern Improvements Suggested
- **React Server Components (RSC):** Maximize the use of Server Components in the `app/` directory. Fetch Sanity data directly in Server Components without `useEffect` or client-side fetchers to improve performance and SEO.
- **Scrollytelling & Animations:** Move away from heavy scroll-event listeners. Utilize `framer-motion` integrated with Intersection Observers (`useInView`, `useScroll`) for performant scrollytelling.
- **Image Optimization:** Enforce strict usage of `next/image` combined with Sanity image builders. Always provide responsive `sizes` attribute and low-quality image placeholders (LQIP) generated by Sanity.
